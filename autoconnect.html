Follow these steps to turn your Debian laptop into a QR-actioned workstation.

1. Install Dependencies

Your laptop needs these tools to watch for files

sudo apt update && sudo apt install -y inotify-tools wget

2. Script Deployment

First, download the script using the button below, then move it to the project folder.

üíæ Download Script 
mkdir -p ~/qr-actions
mv ~/Downloads/bluetooth-listener.sh ~/qr-actions/
chmod +x ~/qr-actions/bluetooth-listener.sh

3. Manual Systemd Configuration

Instead of a script, we will create the service file manually to ensure no errors.

A. Create the Directory

mkdir -p ~/.config/systemd/user/

B. Create the File

Run nano ~/.config/systemd/user/qr-listener.service and paste the following:

[Unit]
Description=QR Actions Bluetooth Listener
After=network.target

[Service]
Type=simple
ExecStart=%h/qr-actions/bluetooth-listener.sh
Restart=always
RestartSec=5

[Install]
WantedBy=default.target

C. Enable Service

systemctl --user daemon-reload
systemctl --user enable qr-listener.service
systemctl --user start qr-listener.service

üîç How the Script Works

The bluetooth-listener.sh is a lightweight daemon that utilizes inotify to watch for filesystem events.

  • The Trigger: It waits for a close_write event in ~/BluetoothDownloads. This happens exactly when a file transfer from your phone completes.
  • PIN Verification: The script reads the second line of the received file. If a PIN is set on your laptop in ~/.config/bluetoothpin.txt, the file's PIN must match, or the script deletes the file immediately.
  • WiFi Automation: If the data starts with WIFI:, it parses the SSID and Password, then uses nmcli to delete any existing old profiles and reconnect to the new network.
  • URL Security: If a URL is detected, it uses wget to download the page as an HTML file into your ~/Downloads folder for offline viewing.
